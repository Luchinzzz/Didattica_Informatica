Concetti ed esempi

Nella programmazione ad oggetti, la definizione di visibilità assegnata alle proprietà delle classi, ai metodi e alle variabili ricopre un ruolo importante. Infatti è proprio grazie a quest'ultimo che possiamo cambiare il significato di un componente di un’applicazione Java tramite i cosiddetti modificatori di accesso. Essi vengono anteposti prima della dichiarazione di un componente, anche più di uno alla volta, in modo da regolarne l'accesso e la visibilità.
  
I principali modificatori di accesso in Java sono:
  1. public: utilizzabile sia per i membri di una classe che per la classe stessa:
    • un attributo o metodo di una classe dichiarato pubblico è accessibile da una qualsiasi classe di un qualsiasi package;
    • una classe pubblica è visibile da un qualsiasi package.
      • In questo esempio possiamo vedere la classe Veicolo che ha un attributo ruote dichiarato pubblico a cui è possibile accedere:
       
       public class Veicolo {
              public int ruote;
      }

            public static void main(String[] args) {
                  Veicolo moto = new Veicolo();
                  moto.ruote = 2;
              
              }
  2. protected: utilizzabile per i membri di una classe:
    • un metodo o attributo di una classe protetto è accessibile all'interno dello stesso package e in tutte le sottoclassi della classe in cui è definito.
      • In questo caso proviamo a modificare l'attributo ruote del nostro nuovo oggetto, ma non ci è possibile poiché ruote è dichiarata come protected ed è stata definita in un altro package:

        package pippo;

        public class Veicolo {
            protected int ruote;
        }

        public class Auto extends Veicolo {
            public void hello() {
                ruote = 4;
            }
        }

        package pluto;
        import pippo.*;

        public static void main(String[] args) {
                Auto a = new auto();
                a.ruote = 20; // // errore di compilazione, campo non accessibile per package diverso
                }   
  3. private: utilizzabile solo con i membri di una classe:
    • un metodo o attributo di una classe privato è accessibile solo all'interno della classe.
      • Qui invece proviamo a modificare l'attributo ruote ereditato da Veicolo, ma non possiamo poiché é privato:

        public class Veicolo {
            private int ruote;
        }
        class Auto extends Veicolo {
            public void hello() {
                
                ruote = 30;// errore di compilazione, variabile non visibile
            }
        } 
  4. default: modificatore applicato quando non ne viene specificato uno:
    • un membro di una classe per default è accessibile solo dalle classi appartenenti al package dove è definito;
    • una classe definita senza il modificatore public è visibile solo dalle classe appartenenti allo stesso package.

Inoltre possiamo definire modificatori di visibilità anche le seguenti parole chiave: 
  1. final: applicabile alle variabili, ai metodi e alle classi:
    • una variabile final diventa una costante, mentre le variabili e i parametri locali non sono modificabili localmente;
    • un metodo final non può essere riscritto in una sottoclasse (no override);
    • una classe final non può essere estesa.
  2. abstract: applicabile a un metodo o a una classe, quest'ultima non può essere istanziata.
  3. static: è possibile applicarlo sia ai membri di una classe che alle variabili:
    • un membro statico è condiviso da tutte le istanze della classe, quindi esso può essere utilizzato senza istanziare quest'ultima. Un esempio di membro statico è la funzione sqrt di Math richiamabile tramite la sintassi Math.sqrt(numero);
    • una variabile di una classe dichiarata statica, essendo condivisa da tutte le istanze della classe, assumerà lo stesso valore per ogni oggetto di una classe. Quindi se un'istanza modifica la variabile statica di conseguenza essa verrà modificata anche in un'altra istanza relativa alla medesima classe.
    • una variabile dichiarata static e public può essere vista come una variabile globale, se aggiungiamo anche il modificatore final quest'ultima diventa una costante globale.

In conclusione le variabili locali sono visibili solo nelle porzioni di codice in cui sono definite e in quelli annidati, con la differenza che non è possibile ridefinire una variabile in quest'ultimo caso.  Mentre i campi e i metodi delle classi sono visibili nella classe stessa e nelle sottoclassi a patto che non abbiano il modificatore private.

  1. Si chiede di creare una  classe Persona la quale è costituita da due variabili di istanza di tipo stringa nome e cognome. Inoltre la classe deve contenere un costruttore parametrico e i metodi get() e set(), si consiglia di prestare particolare attenzione ai modificatori di accesso applicati agli attributi e ai metodi. Che tipo di modificatori sono stati applicati e perché?

La suddetta sottoclasse deve contenere un costruttore parametrico e i metodi set() e get() per le nuove variabili matricola ed esami. Nel caso della classe Persona a cui viene applicato l'incapsulamento, ereditando quest'ultima è possibile per la sottoclasse Studente accedere direttamente ai campi nome e cognome? Motivare la risposta. In caso negativo come è possibile ovviare a questo problema?
